HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage.
  • The style element defines the CSS style for the webpage.
  • The base element defines the URL for the webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • A link element creates the connection to an external file.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A declaration contain two components: The CSS property and the value of the property.
  • The img selector assigns property values to any image.
  • The class attribute selector specifies one or more classnames for an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: Adds the starter code to the current working branch.
  • git commit: Adds a commit message that descibes what the commit contains.
  • git pull origin main: Checks if your local branch is in sync with your base branch.
  • git push origin: To push the changes we have made locally to our remote GitHub branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Arrays are used to store multiple values in a single variable.
  • An if statement will check to see if the condition that we provide is truthy, if it is will perform the function.
  • An Operator is a mathematical symbol that produces a result based on two values.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Functions are a way of packaging functionality that you wish to reuse.
  • Events are code structures that listen to activity in a browser and run code in respone.
